## 📝 문제 요약

N개의 수 A1, A2, ..., AN과 L이 주어진다.

Di = Ai-L+1 ~ Ai 중의 최솟값이라고 할 때, D에 저장된 수를 출력하는 프로그램을 작성

## 💡 접근 방법

- **알고리즘/자료구조: 덱**
- **핵심 아이디어:**
    - 덱에 추가하면서 맨 앞이 최솟값이 되도록 유지

    새로운 값이 들어올 때
    뒤에서부터 현재 값보다 큰 값 제거
    현재 값 추가
    범위를 벗어난 값 제거

    이 과정을 통해 O(N) 으로 해결할 수 있다.

## ✅ 풀이 과정

1. 문제 분석

- 시간 제한: 2.4초
- N≤5,000,000

매번 구간 최소값을 탐색 → O(NL) → 불가능
우선순위 큐 사용 → O(N log N) → 시간 초과 가능

따라서 O(N) 으로 해결해야 한다.

1. 해결 전략 수립
- 덱에 추가하면서 맨 앞이 최솟값이 되도록 유지
- 덱에 인덱스와 값을 함께 넣어 해당범위 비교
    - 현재 추가하려는 값보다 큰 값이 있다면 pollLast()
    - 덱에 현재 값 추가
    - 덱의 맨 앞 출력 ( 인덱스가 범위 밖이면 pollFirst())


## 📌 배운 점 / 주의사항
1. 우선순위 큐 사용 시 시간 초과

PriorityQueue는

- 삽입 : O(log N)
- 삭제 : O(log N)

이 문제는 N이 최대 500만이라
O(N log N) 은 시간 초과가 발생할 수 있다.

→ Deque 사용 → O(N)

2. StringBuilder 사용 시 성능 차이

다음 코드가 시간 초과 발생

sb.append(deque.peekFirst()[1] + " ");

이유
→ + 연산이 문자열 결합을 위해 내부적으로 객체 생성

따라서 다음 방식이 더 빠르다.

sb.append(deque.peekFirst()[1]).append(" ");

문자열 생성 없이 append 체이닝으로 처리된다.
