## 📝 문제 요약

아래와 같이 구성된  미로를 탈출하는데 걸리는 이동 횟수의 최솟값을 구하는 프로그램을 작성하시오.

- 빈 칸: 언제나 이동할 수 있다. ('.')
- 벽: 절대 이동할 수 없다. ('#')
- 열쇠: 언제나 이동할 수 있다. 이 곳에 처음 들어가면 열쇠를 집는다. ('a', 'b', 'c', 'd', 'e', 'f')
- 문: 대응하는 열쇠가 있을 때만 이동할 수 있다. ('A', 'B', 'C', 'D', 'E', 'F')
- 민식이의 현재 위치: 빈 곳이고, 민식이가 현재 서 있는 곳이다. ('0')
- 출구: 달이 차오르기 때문에, 민식이가 가야하는 곳이다. 이 곳에 오면 미로를 탈출한다. ('1')

## 💡 접근 방법

- **알고리즘/자료구조: BFS / 비트마스킹**
- **핵심 아이디어:**
    - BFS 탐색시 각자의 경로에 맞춰서 visited와 키 획득 상태를 따로 관리해야함 → 차원 추가
        
        ```java
        visited[nx][ny][newKey] = true;
        queue.add(new int[] { nx, ny, newKey });
        ```
        
    - 키가 6개 → 비트마스킹으로 관리

# ✅ 풀이 과정

## 1️⃣ 문제 분석

### 📌 문제 핵심 요소

- 시작점: `0`
- 도착점: `1`
- 벽: `#`
- 빈칸: `.`
- 열쇠: `a ~ f` (6개)
- 문: `A ~ F` (해당 열쇠 있어야 통과 가능)

---

1. **단순 BFS가 아니다.**
2. 같은 좌표라도
    - 어떤 열쇠를 가지고 있느냐에 따라
    - 갈 수 있는 경로가 달라진다.

예시:

| 위치 | key 없음 | key 있음 |
| --- | --- | --- |
| (x,y) | 문 A 못 감 | 문 A 통과 가능 |

👉 즉, `(x, y)`만으로 상태를 표현할 수 없다.

---

### 📌 상태 정의

이 문제에서의 상태는:

```
(x, y, keyState)
```

- x : 현재 행
- y : 현재 열
- keyState : 현재 보유한 열쇠 상태

---

### 📌 열쇠 상태 경우의 수

열쇠는 6개

각각 "있다 / 없다"

```
2^6 = 64가지 상태
```

---

## 2️⃣ 해결 전략 수립

### 🎯 전략 핵심

> 좌표 + 열쇠 상태를 함께 관리하는 BFS
> 

---

## 🔥 1. 열쇠 상태 관리 방법

비트마스크 사용

```
intkeyState=0;
```

### 🔑 열쇠 획득

```
keyState|= (1<< (map[nx][ny]-'a'));
```

### 🚪 문 통과 조건

```
if ((keyState& (1<< (map[nx][ny]-'A')))==0)
continue;
```

---

## 🔥 2. 방문 배열 설계

```
boolean[][][]visited=newboolean[N][M][64];
```

의미:

```
visited[x][y][keyState]
```

- 같은 위치라도
- key 상태가 다르면
- 다른 상태로 취급

---

## 🔥 3. BFS 진행 방식

### 큐에 넣을 정보

```
(x,y,keyState)
```

또는

```
(x,y,keyState,distance)
```

---

### BFS 흐름

1. 시작점 `(startX, startY, 0)` 삽입
2. 4방향 탐색
3. 벽이면 skip
4. 열쇠면 keyState 업데이트
5. 문이면 열쇠 검사
6. 방문 안 한 `(nx, ny, newKeyState)`이면 큐 삽입
7. `1`을 만나면 거리 출력
