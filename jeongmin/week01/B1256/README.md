# BOJ 1256 - 사전

## 📌 문제 설명

a n개와 z m개로 이루어진 단어들을 사전순으로 정렬했을 때 k번째 단어를 출력하는 문제

## 💡 해결 아이디어

1. 1번부터 전부 다 사전순으로 정렬하기에는 너무 많다.
2. n+m개의 자리 중에 a가 들어갈 자리 n개를 선택하면 만들 수 있는 단어의 개수를 알 수 있겠다.
3. 사전순으로 봤을 경우 앞자리에 a가 들어가다가 처음으로 z가 나오는 케이스들을 세고 이를 누적하면서 첫 z 전에 a가 몇개인지 알 수 있겠다.
4. 그 이후 사용한 a와 z의 개수를 제외하고 반복하면 되겠군.

## 🧠 코드 해설

```java
dp = new int[201][101];
```

조합 메모리제이션용 dp

```java
			int i = 0;
			
			for (; i <= a; i++) {
				int c = comb(z-1+i, i);
				if (k > c) k -= c;
				else break;
				
				if (i == a) {
					System.out.println(-1);
					return;
				}
			}
```

a-i개의 a가 첫번째 z 앞에 존재할 수 있는지 확인하는 방식  
k가 더 작을 경우 더 적은 a는 불가능하기 때문에 break  
i가 a에 도달했는데도 k값이 더 크다면 주어진 n과 m으로 도달 불가능한 k

```java
			if (i == 0) {
				for (int j = index; j < m+n+1; j++) {
					if (a > 0) {
						output[index++] = 'a';
						a--;
					} else output[index++] = 'z';			
				}
```

i==0일 경우:  
첫번째 z 앞에 a개의 a가 존재할 경우, 즉 aaaazzzz같은 구조이므로 바로 출력

```java
			} else {
				int count = a-i;
				for (int j = 0; j < count; j++) {
					output[index++] = 'a';
					a--;
				}
				output[index++] = 'z';
				z--;
			}
```

아닐 경우 a-i개의 a와 z 1개를 저장하고 다시 처음부터 반복

```java
			if (z == 0) {
				while (index <= m+n) {
					output[index++] = 'a';
				}	
			}
```

만약 z를 다 썼을 경우 남은 자리를 a로 채우고 종료

## 🚀 느낀점

처음에 조합에서 오버플로우 날걸 고려 안해서 좀 헤멨던 것 같다. 조합을 누적하며 키우는 방식이기 때문에 오버플로우가 나기 전에 k 값을 넘을 것이라고 생각했는데 comb(z-1+i, i)가 nCr의 n과 r 모두 1씩 커져서 순식간에 훌쩍 커버린다는 것을 고려하지 못했다.
    
이런 식으로 조합에 최댓값을 지정할 생각을 못해서 너무 큰 조합을 계산하는 일이 없도록 이런 방식을 택한 것인데 결국 최댓값을 지정한 걸 보면 그냥 앞에서부터 1개씩 하는게 더 쉽고 편했을 것 같다.
