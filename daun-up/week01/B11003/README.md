### 문제 분석

- 최솟값을 찾는다. Di = Ai-L+1 ~ Ai 중의 최솟값이라고 할 때, D에 저장된 수를 출력하는 프로그램을 작성하시오. 이때, i ≤ 0 인 Ai는 무시하고 D를 구해야 한다.
- 범위 (1 ≤ L ≤ N ≤ 5,000,000)
- N 이 500만 개이다. 매번 L 만큼 N 개 탐색하면 O(N _ L) 최악일 때 5 000 000 _ 5 000 000 이면 25 000 000 000 000 25조의 연산...

### 문제 접근

- 윈도우 슬라이딩!
- Priority Queue를 사용하는 방법도 생각할 수 있다. 최솟값 조회는 peek()로 O(1) 에 가능하지만, 삽입과 삭제 연산이 O(log L) 이기 때문에 전체 시간 복잡도는 O(N log L)
- Priority Queue는 항상 전체 원소를 힙 구조로 유지해야 하기 때문에 log 비용이 발생한다.
- 각 원소를 (값, 인덱스) 쌍으로 저장해서 top 이 윈도우 인덱스 범위 밖이면 poll 로 제거한다. 특정 원소를 직접 찾아 삭제하는 경우 remove() 연산이 O(L) 이기 때문에 잘못 구현하면 **O(NL)** 이 될 위험도 있다.

### Deque

- Deque (Double Ended Queue) 는 앞/뒤 양쪽에서 삽입과 삭제가 가능한 자료구조

- Priority Queue는 항상 전체 원소를 힙 구조로 유지해야 하기 때문에 log 비용이 발생한다.
- Deque 를 사용하여 현재 윈도우에서 최솟값이 될 가능성이 있는 값만 유지한다. (계속 계속 최솟값만을 남기는 방식)
- Deque 에서는 앞에서 뒤로 값이 오름차순으로 유지된다. 새로 들어오는 값이 더 작으면 뒤쪽(꼬리)에서 큰 값들을 제거해서 그 오름차순을 유지한다.
- 즉, 핵심은 Deque 를 값 기준 오름차순으로 유지하는 것이다.
- Deque 의 머리에 항상 현재 윈도우의 최솟값 후보가 위치하므로 peekFirst() 로 최소값을 즉시 출력할 수 있다. 이를 위해 새 원소가 들어올 때마다 Deque 의 꼬리에서 새 원소 보다 큰 값들은 앞으로 어떤 윈도우에서도 최소가 될 수 없으므로 제거한다.

1. 꼬리에서 큰 값 제거
2. 현재 값 삽입
3. 머리에서 범위 밖 인덱스 제거를 수행

### 뒤에서 현재 값 보다 큰 값 제거

- 나는 이 부분이 이해가 가장 안 갔다. 예시를 들어보면 이런 느낌
- 덱은 윈도우 안에서 최소값이 될 가능성이 있는 후보들만 저장 (윈도우 ⊇ 덱)

```
{5, 4, 3, 2, 1), L = 3
i = 0 x = 5 dq = {5} 윈도우 [5]
i = 1 x = 4 dq = {4} 윈도우 [5,4]
i = 2 x = 3 dq = {3} 윈도우 [5,4,3]
i = 3 x = 2 dq = {2} 윈도우 [4,3,2]
i = 4 x = 1 dq = {1} 윈도우 [3,2,1]
(매번 새 값이 더 작아서 뒤에서 큰 값이 다 날아감)

{1, 2, 3, 4, 5}, L = 3
i = 0 x = 1 dq = {1} 윈도우 [1]
i = 1 x = 2 dq = {1,2} 윈도우 [1,2]
i = 2 x = 3 dq = {1,2,3} 윈도우 [1,2,3]
i = 3 x = 4 dq = {2,3,4} 윈도우 [2,3,4]
i = 4 x = 5 dq = {3,4,5} 윈도우 [3,4,5]
(뒤 제거 거의 없음 → dq가 커지다가 앞에서 윈도우 밖 제거로 유지)
```

```java
while (!dq.isEmpty() && A[dq.peekLast()] > A[i])
    dq.pollLast();
```

### 윈도우 범위를 벗어난 인덱스 제거

```java
[i - L + 1 , i] // 윈도우 범위
if (dq.peekFirst() <= i - L)
    dq.pollFirst();
```

### 덱의 맨 앞이 현재 윈도우의 최솟값!!

```java
A[dq.peekFirst()]
```

### 시간 복잡도

- 각 원소는 덱에 한 번 들어가고 최대 한 번 제거하기 때문에 전체 시간 복잡도는 O(N) 이 된다.

| 방법                    | 시간 복잡도 |
| ----------------------- | ----------- |
| 브루트포스              | O(NL)       |
| Priority Queue          | O(N log L)  |
| Deque (Monotonic Queue) | **O(N)**    |

| 자료구조      | 정렬 기준     |
| ------------- | ------------- |
| Queue         | 입력 순서     |
| Deque         | 앞/뒤 자유    |
| PriorityQueue | 값의 우선순위 |
