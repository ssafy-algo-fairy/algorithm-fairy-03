🐼 욕심쟁이 판다(B1937) 풀이 발표

🌳 문제 핵심판다가 이동할 때마다 현재 칸보다 대나무가 더 많은 칸으로만 이동하며, 최대한 많은 칸을 방문해야 하는 문제입니다.모든 칸($N \times N$)에서 출발해 볼 수 있지만, 단순 DFS로 탐색할 경우 중복 방문으로 인해 **시간 초과($O(4^{N^2})$)**가 발생합니다. 따라서 이미 계산된 경로의 길이를 저장하는 DP가 필수적입니다.

💡 핵심 알고리즘 : DFS + 메모이제이션 (DP)
탐색 전략 = 모든 칸을 순회하며 해당 칸에서 시작하는 최장 경로를 탐색
  └─ 현재 칸 (y, x) 방문
     ├─ 이미 방문하여 최댓값이 계산된 칸(dp[y][x] != -1)이라면? 즉시 반환 (Memoization)
     └─ 처음 방문하는 칸이라면? 
          ├─ 상하좌우 4방향 탐색
          ├─ 조건: 격자 내부이면서 + 다음 칸의 대나무가 현재보다 많아야 함 (arr[ny][nx] > arr[y][x])
          ├─ 다음 칸들 중 가장 멀리 갈 수 있는 경로의 길이 + 1(현재 칸)을 선택
          └─ 결과값을 dp[y][x]에 저장 후 반환

```
static int dfs(int y, int x) {
    // 1. 이미 계산된 결과가 있다면 바로 반환 (메모이제이션)
    if (dp[y][x] != -1) return dp[y][x];

    // 2. 최소 한 칸(현재 칸)은 방문하므로 1로 초기화
    int next = 1;

    for (int i = 0; i < 4; i++) {
        int ny = y + dy[i];
        int nx = x + dx[i];

        // 격자 범위 확인
        if (ny < 0 || ny >= N || nx < 0 || nx >= N) continue;

        // 3. 현재 칸보다 대나무가 더 많은 곳으로만 이동 가능
        if (arr[ny][nx] > arr[y][x]) {
            // 주변 칸들 중 최댓값을 갱신 (재귀 호출)
            next = Math.max(next, dfs(ny, nx) + 1);
        }
    }

    // 4. 최종 계산된 값을 dp 배열에 저장 (기록)
    return dp[y][x] = next;
}
```


항목,내용
탐색 방식,DFS (깊이 우선 탐색)
최적화 기법,메모이제이션 (Dynamic Programming)
이동 조건,arr[next] > arr[current] (엄격한 오름차순)
시간 복잡도,O(N2) (각 칸을 한 번씩만 계산하면 끝남)

🔑 Key Point:모든 출발점에서 DFS를 시도하더라도, dp 배열이 -1이 아닌 값을 가지고 있다면 더 이상 깊이 들어가지 않고 바로 값을 리턴합니다. 이 덕분에 무수히 많은 중복 연산을 획기적으로 줄여 $500 \times 500$ 크기의 격자에서도 매우 빠르게 동작합니다.


🎯 결론 및 배울 점중복 연산 제거의 위력: 단순 재귀는 지수 시간 복잡도를 가지지만, 메모이제이션을 결합하는 순간 선형 시간 복잡도에 가까워지는 것을 확인할 수 있습니다.오름차순 이동의 특성: 이동 조건이 항상 대나무가 "많은" 곳으로만 향하기 때문에 별도의 visited 배열 없이도 사이클(Cycle)이 절대 발생하지 않는 구조임을 파악하는 것이 중요합니다. (무조건 더 큰 숫자로만 가기 때문에 다시 돌아올 수 없음)
