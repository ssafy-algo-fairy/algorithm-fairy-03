# 🌙 달이 차오른다, 가자. (1194) 풀이 발표

---

## 🌳 문제 핵심

> 미로를 탈출하기 위해 단순히 최단 경로를 찾는 것을 넘어, **'열쇠의 획득 상태'**에 따라 방문할 수 있는 곳이 달라지는 문제입니다. 
> 같은 좌표에 있더라도 어떤 열쇠를 가졌느냐에 따라 미래의 경로가 결정되므로, **좌표 (y, x)에 비트마스킹을 이용한 열쇠 상태(key)를 추가한 3차원 방문 배열**이 핵심입니다.

---

## 💡 핵심 알고리즘 : BFS + 비트마스킹 (Bitmasking)

### 핵심 코드
```java
    static int solve(){
        int dx[] = {0, 0, 1, -1};
        int dy[] = {1, -1, 0, 0};

        while(!que.isEmpty()){ // 큐가 빌 때까지 탐색
            Node now = que.poll();
            
            for(int i=0; i<4; i++){
                int ny = now.y + dy[i];
                int nx = now.x + dx[i];
                int nowKey = now.key; 
                
                if(!check(ny,nx)) continue;
                if(arr[ny][nx] == '#') continue; // 벽 통과 불가
                
                // 1. 탈출구('1') 도착 시 즉시 종료 (BFS이므로 최단거리 보장)
                if(arr[ny][nx] == '1') return now.count+1;

                // 2. 열쇠('a'-'f')를 만난 경우: 비트 연산 OR(|)로 열쇠 추가
                if('a' <= arr[ny][nx] && arr[ny][nx] <= 'f'){
                    nowKey = nowKey | (1 << (arr[ny][nx] - 'a'));
                }
                
                // 3. 문('A'-'F')을 만난 경우: 비트 연산 AND(&)로 해당 열쇠 보유 확인
                else if('A' <= arr[ny][nx] && arr[ny][nx] <= 'F'){
                    if((nowKey & (1 << (arr[ny][nx] - 'A'))) == 0) continue;
                }

                // 4. 방문 체크: 현재 내가 가진 '열쇠 조합'으로 이 좌표에 온 적이 있는가?
                if(visited[ny][nx][nowKey]) continue;

                visited[ny][nx][nowKey] = true;
                que.add(new Node(ny, nx, nowKey, now.count+1));
            }
        }
        return -1; // 탈출 불가능한 경우
    }
```

항목,내용
탐색 방식 : BFS (Breadth-First Search) - 가중치가 1인 그래프에서 최단 거리 보장

상태 관리 : 비트마스킹 (Bitmasking) - 6개의 열쇠 유무를 0~63(26)의 정수로 표현

방문 배열 : "visited[N][M][64] - (세로, 가로, 열쇠상태) 3차원 배열로 중복 방문 방지"

이동 조건 : "빈 칸('.')이나 시작점('0')은 자유 이동, 열쇠는 습득, 문은 열쇠 있을 때만 통과"



🔑 Key Point: 3차원 방문 처리와 비트마스킹
1. 왜 3차원인가?
일반적인 2차원 방문 체크는 "열쇠를 얻기 위해 이전에 지나온 길을 다시 돌아가는 상황"을 막아버립니다. 열쇠를 새로 얻는 순간 그것은 완전히 새로운 상태의 탐색이 되어야 하므로, 열쇠 보유 현황별로 별도의 지도(Layer)를 갖는다고 생각해야 합니다.

2. 비트마스킹의 효율성
열쇠가 6개일 때, boolean[] keys = new boolean[6]을 매 노드마다 들고 다니는 것보다, 1 << 6 (64) 크기의 비트로 관리하는 것이 메모리나 비교 연산(&, |) 속도 면에서 훨씬 압도적입니다.

🎯 결론 및 배울 점
- 상태 공간의 확장: 문제에서 요구하는 조건(열쇠)이 단순 좌표 방문에 영향을 줄 때, 방문 배열의 차원을 확장하여 상태를 분리하는 법을 배웠습니다.

- BFS의 본질: BFS는 모든 가중치가 동일할 때 "먼저 도달하는 것이 곧 최단거리"라는 특성을 가집니다. 이를 비트마스킹과 결합해 복잡한 조건부 최단 경로 문제를 해결할 수 있었습니다.

- 엣지 케이스 처리: 시작 지점('0')이 여러 개가 아니라는 점, 탈출구('1')가 여러 개일 수 있다는 점 등을 고려하여 return 시점을 잡는 디테일을 익혔습니다.
