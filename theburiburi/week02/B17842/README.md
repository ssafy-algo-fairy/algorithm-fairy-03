# 🔍 [백준 17842] 버스 노선 문제 풀이 분석

---

## 🌳 문제 핵심

> **트리(Tree) 구조**에서 모든 정점을 지나는 최소 개수의 버스 노선(경로)을 구하는 문제입니다.
> 1. **제한 사항**: 정점 개수 N과 N-1개의 간선이 주어지며, 이는 사이클이 없는 전형적인 트리 형태입니다.
> 2. **해결 전략**: 트리의 모든 노드를 덮기 위한 최소 경로 수는 트리의 끝점, 즉 **리프 노드(Leaf Node)**를 짝지어 연결하는 것과 같습니다.
> 3. **효율성**: 복잡한 그래프 탐색 없이 각 정점의 **차수(Degree)**만 계산하여 리프 노드의 개수를 도출하는 수학적/아이디어 기반 최적화가 필요합니다.

---

## 💡 핵심 알고리즘: 트리(Tree) 차수 계산 및 접근

```text
알고리즘 흐름 = 간선 정보 입력 ➔ 각 정점의 차수 계산 ➔ 리프 노드 개수 확인 ➔ 수식 적용
 └─ 정점 차수(Degree) 계산
     └─ 배열(arr)을 활용해 두 정점이 연결될 때마다 각각의 차수를 1씩 증가
     └─ 모든 간선 입력 후, 차수가 1인 정점(리프 노드)을 판별
 └─ 최소 경로 수학적 도출
     ├─ 개념: 리프 노드 2개를 양 끝점으로 삼아 경로를 만들면 중간 노드들이 자연스럽게 포함됨
     ├─ 계산: 리프 노드를 2개씩 짝지으면 최소 노선이 되므로, (리프 노드 개수 + 1) / 2가 정답
```

```
// 1. 차수 계산 및 리프 노드 판별
arr = new int[N];
for(int i = 1; i < N; i++){
    st = new StringTokenizer(br.readLine());
    int left = Integer.parseInt(st.nextToken());
    int right = Integer.parseInt(st.nextToken());
    
    // 무방향 간선이므로 양쪽 정점의 차수(Degree) 증가
    arr[left]++;
    arr[right]++;
}

// 2. 리프 노드(차수가 1인 정점) 개수 카운트
int count = 0;
for(int i = 0; i < N; i++){
    if(arr[i] == 1) count++;
}

// 3. 최소 버스 노선 개수 출력: 리프 노드를 2개씩 짝지음 (홀수 대비 +1 후 나누기 2)
System.out.println((count + 1) / 2);
```

항목,내용
자료구조,1차원 배열 (Array)
탐색 방식,단순 반복을 통한 차수(Degree) 확인
시간 복잡도,O(N) (간선 입력 및 배열 1회 순회)
공간 복잡도,O(N) (정점의 차수를 저장하기 위한 배열 공간)

🎯 결론 및 배울 점

문제의 본질 파악: 복잡한 트리 탐색(DFS/BFS) 알고리즘을 무작정 사용하지 않고도, '트리의 모든 정점을 덮으려면 끝점(리프 노드)끼리 연결하면 된다'는 구조적 특징을 파악하여 단순한 식으로 치환했습니다.

입출력 최적화: 많은 양의 노드 데이터가 들어올 수 있으므로 BufferedReader와 StringTokenizer를 사용하여 입력 처리 속도를 높였습니다.

직관적인 접근: 리프 노드들을 한 번의 경로로 최대한 많이 포함시키기 위해 2개씩 짝지어 연결하는 논리가 핵심적으로 작용했습니다.
