# 🔍 [백준 4991] 로봇 청소기 문제 풀이 분석

---

## 🌳 문제 핵심

> **2차원 격자(w × h)** 형태의 방에서 로봇 청소기가 가구('x')를 피해 모든 더러운 칸('*')을 청소하는 최소 이동 횟수를 구하는 문제입니다.
> 1. **제한 사항**: 방의 크기는 최대 20x20이며, 더러운 칸의 개수는 **최대 10개**입니다.
> 2. **해결 전략**: 먼지의 개수가 10개 이하로 매우 적다는 점이 핵심입니다. 방문해야 할 목표 지점이 여러 개일 때, 방문 상태를 기록하기 위해 **비트마스킹(Bitmasking)**을 결합한 BFS 탐색을 사용해야 합니다.
> 3. **상태 관리**: 단순히 위치(y, x)만으로는 어떤 먼지를 청소했는지 알 수 없으므로, 3차원 방문 배열 `visited[y][x][먼지청소상태(bit)]`를 활용해 상태를 분리하여 탐색합니다.

---

## 💡 핵심 알고리즘: BFS (너비 우선 탐색) + Bitmasking (비트마스킹)

```text
알고리즘 흐름 = 방 구조 입력 및 먼지 좌표 매핑 ➔ 비트마스크 BFS 탐색 ➔ 최단 거리 반환
 └─ 먼지 좌표 식별 (Mapping)
     └─ 각 먼지의 좌표(y, x)를 고유 키로 변환하여 0부터 `dustCnt-1`까지의 번호(ID)를 부여 (HashMap 활용)
 └─ BFS + 비트마스킹 탐색
     ├─ 큐(Queue)에 `[y, x, 현재 청소한 먼지 bit, 이동 횟수]` 저장
     ├─ 이동할 칸이 먼지('*')인 경우: 해당 먼지의 ID를 확인하고 비트 연산(`|`)을 통해 청소 상태 갱신
     ├─ 종료 조건: 현재 비트가 `(1 << dustCnt) - 1` (모든 먼지를 다 치운 상태)가 되면 이동 횟수 반환

```
// 1. 먼지 좌표에 고유 번호(ID) 부여를 위한 매핑
if(arr[i][j] == '*'){
    // makeKey(i, j) = y * 100 + x 로 고유값 생성
    map.put(makeKey(i, j), dustCnt++); 
}

// 2. 비트마스킹을 적용한 3차원 방문 배열 및 BFS
// h: 세로, w: 가로, 1<<dustCnt: 먼지 청소 상태 조합 (최대 2^10 = 1024)
boolean visited[][][] = new boolean [h][w][1<<dustCnt];

while(!que.isEmpty()){
    // ... (4방향 탐색 로직) ...
    
    // 3. 이동할 위치가 먼지인 경우 상태 비트 갱신
    if(arr[ny][nx] == '*'){
        int num = map.get(makeKey(ny, nx)); // 먼지의 고유 번호 획득
        if((bit & (1<<num)) == 0){ // 아직 청소하지 않은 먼지라면
            bit = bit | (1<<num);  // 청소 처리 (비트 켜기)
        }
    }
    else if(arr[ny][nx] == 'x') continue; // 가구(장애물) 통과 불가

    // 4. 모든 먼지를 다 청소했는지 확인
    if(bit == (1 << dustCnt)-1){
        return now[3] + 1; // 최소 이동 횟수 반환
    } 

    // 방문 체크 및 큐 삽입
    if(visited[ny][nx][bit]) continue;
    que.add(new int[]{ny, nx, bit, now[3]+1});
    visited[ny][nx][bit] = true;
}
```

항목,내용
자료구조,"Queue, HashMap, 3차원 배열(Array)"
탐색 방식,BFS (너비 우선 탐색) + Bitmasking
시간 복잡도,"O(W × H × 2^D) (D는 먼지의 수, 최대 10)"
공간 복잡도,O(W × H × 2^D) (3차원 방문 배열 및 큐 메모리)

🎯 결론 및 배울 점

비트마스킹의 활용: 여러 타겟(최대 10개)의 방문 여부를 독립적으로 관리해야 할 때, 배열 대신 비트 연산을 사용해 visited 배열의 차원을 줄이고 메모리와 시간을 획기적으로 절약했습니다.

HashMap을 통한 좌표-ID 매핑: 각 먼지의 좌표 (y, x)를 y * 100 + x 수식을 이용해 고유한 정수 키로 변환하고 해시맵에 저장함으로써, 특정 좌표의 먼지가 몇 번째 비트인지 O(1)에 빠르게 찾을 수 있도록 구현한 점이 돋보입니다.

다중 테스트 케이스 최적화: w == 0 && h == 0 종료 조건에 맞춰 변수들과 자료구조를 매번 올바르게 초기화하는 구조가 잘 짜여 있습니다.
