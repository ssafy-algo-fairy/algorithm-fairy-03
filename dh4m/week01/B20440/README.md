## 문제 요약

* **문제명**: 백준 20440번 - 🎵니가 싫어 싫어 너무 싫어 싫어 오지 마 내게 찝쩍대지마🎵
* **목표**: $N$마리의 모기가 방에 들어오고 나간 시간 범위 $[S, E)$가 주어질 때, 모기가 가장 많이 있었던 시간대의 최대 모기 수와 그 상태가 유지된 가장 빠른 시간 구간 $[T_s, T_e)$를 구하는 문제입니다.
* **제한 사항**: 모기의 수 $N$은 최대 1,000,000이며, 시간 범위는 0부터 2,100,000,000까지입니다. 구간의 끝점 $E$는 포함되지 않음에 유의해야 합니다.

---

## 사용 로직 및 핵심 알고리즘

* **스위핑 (Sweeping) 알고리즘**: 모든 모기의 입실 시점($+1$)과 퇴실 시점($-1$)을 독립적인 이벤트로 분리하여 리스트에 저장한 뒤, 시간순으로 정렬하여 순차적으로 처리합니다.
* **좌표 압축 개념 활용**: 실제 시간의 범위가 매우 크기 때문에 모든 시간을 배열로 관리할 수 없습니다. 따라서 유의미한 변화가 발생하는 시점(입실, 퇴실 시간)들만 추출하여 정렬함으로써 효율적인 탐색을 수행합니다.
* **구간 관리**: 현재 모기 수(`mos`)가 최대치(`max_mos`)를 갱신할 때 시작 시점을 기록하고, 최대치 상태가 끝나고 감소하는 시점에 종료 시점을 확정하여 최장의 연속된 최대 구간을 도출합니다.

## 코드 하이라이트

```java
// 모든 이벤트를 시간순으로 정렬
times.sort((a, b) -> a.t - b.t);

int now = 0;
boolean now_max = false;
int max_start = 0, max_end = 0, max_mos = 0, mos = 0;

for (int i = 0; i < 2 * n; i++) {
    now = times.get(i).t;
    
    // 동일한 시간에 발생하는 모든 이벤트를 한꺼번에 처리하여 현재 모기 수 계산
    while (i < 2 * n && times.get(i).t == now)
        mos += times.get(i++).mos;
    i--; // while문에서 증가한 인덱스 보정

    // 최대 모기 수 갱신 시
    if (mos > max_mos) {
        max_mos = mos;
        max_start = now;
        now_max = true; // 현재 최대 밀집 상태임을 표시
    }
    // 최대 상태였으나 모기 수가 줄어들기 시작하는 시점
    else if (mos < max_mos && now_max) {
        now_max = false;
        max_end = now; // 최대 상태가 끝나는 지점 기록
    }
}

```

## 시간 복잡도 분석

* **데이터 수집 및 객체 생성**: $N$개의 입력에 대해 $2N$개의 `Time` 객체를 생성하므로 $O(N)$입니다.
* **정렬**: $2N$개의 이벤트를 정렬하는 데 $O(2N \log 2N)$이 소요됩니다. 이는 전체 로직 중 가장 큰 비중을 차지합니다.
* **스위핑**: 정렬된 리스트를 한 번 순회하며 처리하므로 $O(N)$입니다.
* **총합**: 최종 시간 복잡도는 $O(N \log N)$입니다. $N=1,000,000$일 때 약 $2 \times 10^7$ 정도의 연산이 예상되므로 2초의 제한 시간 내에 안정적으로 통과 가능합니다.

## 학습 포인트 및 개선 사항

* **동일 시간 이벤트 처리**: `while` 루프를 사용하여 같은 시각에 들어오고 나가는 모기들을 합산하여 처리함으로써, 순간적으로 `max_mos`를 초과했다가 바로 내려가는 경우를 정확하게 배제할 수 있었습니다.
* **종료 구간 설정**: 문제 조건에서 요구하는 '가장 빠른 최대 구간'을 찾기 위해, `now_max`라는 플래그를 활용하여 최대치가 처음 달성된 시점(`max_start`)과 그 상태가 처음 깨지는 시점(`max_end`)을 포착하는 로직이 효율적이었습니다.
* **메모리 최적화**: 현재 `ArrayList<Time>`을 사용하여 객체 생성 비용이 발생합니다. 메모리 사용량을 줄여야 할 경우, 시간값과 변화량을 각각의 1차원 배열(`int[]`)에 담아 정렬하거나 `TreeMap`을 활용하는 방식을 고려할 수 있습니다.
