# 백준 1194 - 달이 차오른다, 가자.

## 1. 문제 요약

미로에 갇힌 민식이가 탈출하기 위한 최소 이동 횟수를 구하는 문제입니다.

* **출발점('0')**에서 시작하여 **탈출구('1')**에 도달해야 합니다.
* 미로에는 빈 공간('.'), 벽('#'), 열쇠('a'-'f'), 문('A'-'F')이 존재합니다.
* 대응하는 열쇠가 있어야만 문을 열고 지나갈 수 있습니다.
* 한 번 얻은 열쇠는 계속 보유하며, 같은 자리를 방문하더라도 **보유한 열쇠의 상태가 다르면** 다른 상태로 간주하여 방문할 수 있습니다.

## 2. 사용 로직 및 핵심 알고리즘

이 문제는 가중치가 없는 그래프에서의 최단 거리를 찾는 문제이므로 **BFS(Breadth-First Search)** 알고리즘을 사용합니다. 다만, 열쇠의 보유 상태에 따라 방문 여부가 달라지므로 **비트마스킹(Bitmasking)**을 활용한 상태 관리가 핵심입니다.

* **상태 정의**: `State` 클래스를 통해 현재 좌표 `(i, j)`와 현재 보유한 열쇠의 비트마스크 `key`를 관리합니다.
* **방문 체크**: `visit[n][m][1 << 6]` 3차원 배열을 사용하여 특정 좌표에 어떤 열쇠 조합을 가지고 방문했는지 기록합니다.
* **비트 연산**:
* 열쇠 습득: `key |= (1 << (map[di][dj] - 'a'))`
* 문 확인: `(key & (1 << (map[di][dj] - 'A'))) != 0`



## 3. 코드 하이라이트

```java
// 3차원 방문 배열: [행][열][열쇠 보유 상태(비트마스크)]
visit = new boolean[n][m][1 << 6];

// BFS 탐색 루프
while (!q.isEmpty()) {
    State now = q.poll();
    
    for (int[] d : delta) {
        int di = now.i + d[0];
        int dj = now.j + d[1];
        int key = now.key;

        if (isValid(di, dj)) {
            char cell = map[di][dj];
            
            // 1. 빈 공간 또는 이동 가능한 상태
            if (cell == '.') {
                if (!visit[di][dj][key]) {
                    visit[di][dj][key] = true;
                    q.add(new State(di, dj, key));
                }
            }
            // 2. 열쇠 발견: 비트 연산으로 열쇠 추가
            else if ('a' <= cell && cell <= 'f') {
                int nextKey = key | (1 << (cell - 'a'));
                if (!visit[di][dj][nextKey]) {
                    visit[di][dj][nextKey] = true;
                    q.add(new State(di, dj, nextKey));
                }
            }
            // 3. 문 발견: 비트 연산으로 열쇠 보유 여부 확인
            else if ('A' <= cell && cell <= 'F') {
                if ((key & (1 << (cell - 'A'))) != 0) {
                    if (!visit[di][dj][key]) {
                        visit[di][dj][key] = true;
                        q.add(new State(di, dj, key));
                    }
                }
            }
            // 4. 탈출구 도달
            else if (cell == '1') {
                complete = true;
                break bfsLoop;
            }
        }
    }
}

```

## 4. 시간 복잡도 분석

* **상태 공간의 크기**: 미로의 크기 $N \times M$에 열쇠의 상태 수 $2^6(64)$을 곱한 $N \times M \times 64$가 전체 상태의 개수입니다.
* **탐색 비용**: 각 상태에서 상하좌우 4방향을 탐색하므로, 전체 시간 복잡도는 $O(N \times M \times 2^K)$입니다 (이때 $K$는 열쇠의 개수인 6).
* 이 문제의 최대 입력값($50 \times 50$)을 대입하면 약 160,000개의 상태를 확인하게 되므로, 제한 시간 내에 충분히 통과 가능합니다.

## 5. 학습 포인트 및 개선 사항

* **비트마스킹의 활용**: 단순히 방문 여부를 `boolean[N][M]`으로 처리할 수 없을 때, 상태(State)를 비트로 압축하여 차원을 늘리는 기법을 익힐 수 있었습니다.
* **BFS 레벨 관리**: `len` 변수를 통해 이동 거리를 계산할 때, 큐의 사이즈(`qsize`)만큼 반복문을 돌려 레벨 단위로 탐색하는 구조를 취하여 최단 거리를 정확히 계산했습니다.
* **개선 사항**: 현재 코드에서는 `map[di][dj] == '.'`인 경우만 처리하고 있는데, 실제로는 시작점('0')도 빈 공간으로 처리해야 합니다. 제출된 코드에서는 입력 시 '0'을 '.'으로 변경하여 예외를 방지한 점이 효율적입니다.
