### 📌 문제 정보

- **번호:** 1230  
- **제목:** 문자열 거리  
- **난이도:** Gold 1
- **분류:** 다이나믹 프로그래밍(DP), 문자열

---

### 💡 접근 방식

> 문자열 `O`를 문자열 `N`과 같게 만들기 위해 필요한 **문자열 삽입 횟수의 최소값**을 구하는 문제입니다.  
> 삽입은 문자 하나가 아니라 **연속된 문자열 덩어리 전체를 한 번에 삽입**할 수 있다는 점이 핵심입니다.

---

### 🔹 1단계 – DP 정의

`dp[i][j]`

- `O[i]`를 `N[j]`에 매칭했을 때
- `O[0..i]`를 `N[0..j]` 안에 맞추는 최소 삽입 횟수

즉, **마지막 문자가 반드시 `O[i] ↔ N[j]`로 대응된 상태**를 의미함

---

### 🔹 2단계 – 첫 문자 초기화

첫 문자 `O[0]`를 `N[j]`에 맞추는 경우:

- `j == 0` → 앞에 삽입 없음 → `0`
- `j > 0` → `N[0..j-1]` 전체를 한 번에 삽입 → `1`

```java
for (int j = 0; j < N.length(); j++) {
    if (o[0] == n[j]) {
        dp[0][j] = (j == 0) ? 0 : 1;
    }
}
```

---

### 🔹 3단계 – DP 전이

`O[i] == N[j]`일 때 두 가지 경우가 존재

#### 1️⃣ 바로 이어서 매칭

이전 문자 `O[i-1]`가 `N[j-1]`에 매칭된 경우

```
O[i-1] → N[j-1]
O[i]   → N[j]
```

중간에 비는 문자가 없으므로 삽입 구간 추가 없음

```java
dp[i][j] = dp[i-1][j-1];
```

---

#### 2️⃣ 더 앞에서 건너뛰어 매칭

이전 문자 `O[i-1]`가 `N[k] (k ≤ j-2)`에 매칭된 경우

```
O[i-1] → N[k]
O[i]   → N[j]
```

`N[k+1..j-1]`가 삽입 문자열이 되므로 삽입 1회 추가

```java
dp[i][j] = prefixMin + 1;
```

여기서 `prefixMin`은  
`dp[i-1][0..j-2]` 중 최소값을 의미함

---

### 🔹 4단계 – 마지막 구간 처리

`O`의 마지막 문자가 `N[j]`에 매칭되었을 때

- `j == N.length()-1` → 뒤에 삽입 없음
- `j < N.length()-1` → 뒤에 남은 부분을 한 번에 삽입 → `+1`

```java
int cur = dp[O.length()-1][j];
if (j != N.length()-1) cur++;
```

---

### 💻 핵심 코드

```java
if (o[i] == n[j]) {
    if (dp[i-1][j-1] != Integer.MAX_VALUE) {
        dp[i][j] = Math.min(dp[i][j], dp[i-1][j-1]);
    }

    if (prefixMin != Integer.MAX_VALUE) {
        dp[i][j] = Math.min(dp[i][j], prefixMin + 1);
    }
}
```

---

### ⏳ 복잡도 분석

- **시간 복잡도:** `O(|O| × |N|)`
- **공간 복잡도:** `O(|O| × |N|)`

문자열 길이가 최대 1000이므로 충분히 계산 가능합니다.
