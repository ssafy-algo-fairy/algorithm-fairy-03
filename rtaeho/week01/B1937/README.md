### 📌 문제 정보

- **번호:** 1937  
- **제목:** 욕심쟁이 판다  
- **난이도:** Gold 3  
- **분류:** DFS, DP(메모이제이션), 그래프 탐색

---

### 💡 접근 방식

> 판다는 현재 위치보다 대나무가 더 많은 칸으로만 이동할 수 있습니다.  
> 즉, 이동 방향이 항상 **값이 증가하는 방향**이기 때문에 사이클이 발생하지 않습니다.  
> 따라서 각 칸에서 시작했을 때 이동할 수 있는 최대 칸 수를 **DFS + 메모이제이션(DP)** 으로 계산했습니다.

---

### 🔹 1단계 – 상태 정의

- `dp[r][c]` : `(r, c)`에서 시작했을 때 이동할 수 있는 최대 칸 수
- 처음 방문한 칸이라면 DFS로 값을 계산하고, 이후에는 저장된 값을 재사용

---

### 🔹 2단계 – DFS 탐색

현재 칸 `(r, c)`에서 상하좌우를 확인하면서 현재 칸보다 대나무가 많은 칸으로만 이동

```java
if (map[nr][nc] > map[r][c]) {
    dp[r][c] = Math.max(dp[r][c], dfs(nr, nc) + 1);
}
```

---

### 🔹 3단계 – 메모이제이션

- 이미 계산된 칸은 다시 탐색하지 않고 바로 반환

```java
if (dp[r][c] != 0) {
    return dp[r][c];
}
```

---

### 💻 핵심 로직

#### DFS + DP

```java
static int dfs(int r, int c) {
    if (dp[r][c] != 0) return dp[r][c];

    dp[r][c] = 1;

    for (int d = 0; d < 4; d++) {
        int nr = r + dr[d];
        int nc = c + dc[d];

        if (nr < 0 || nr >= N || nc < 0 || nc >= N) continue;

        if (map[nr][nc] > map[r][c]) {
            dp[r][c] = Math.max(dp[r][c], dfs(nr, nc) + 1);
        }
    }

    return dp[r][c];
}
```

---

### ⏳ 복잡도 분석

- **시간 복잡도:** `O(N^2)`
    - 각 칸은 DFS로 한 번만 계산됨
    - 각 칸에서 최대 4방향 탐색

- **공간 복잡도:** `O(N^2)`
    - `map`, `dp` 배열 사용

---

### ⚠️ 어려웠던 점

- 처음에는 그냥 DFS로 다 탐색하려고 했지만, 같은 칸을 계속 탐색하게 되어 시간 초과가 발생했습니다.
- 이후 각 칸에서 시작했을 때의 최대 이동 횟수를 저장하는 DP 배열을 활용해 중복 탐색을 제거하는 방식으로 해결했습니다.
- SQLD 준비하느라 다른 사람 풀이 좀 봤습니다.
