### 💡 접근 방식

> 모기의 존재 구간이 `[TE, TX)`로 주어질 때, 동시에 존재하는 모기 수의 최댓값과 그 최댓값이 유지되는 연속 구간을 구하는 문제입니다.  
> 입장/퇴장 시간을 각각 정렬한 뒤, 투포인터로 시간 흐름을 스위핑하며 현재 모기 수를 갱신했습니다.

---

### 🔹 1단계 – 입장/퇴장 시간 분리 및 정렬

- 입장 시간은 `in[]`, 퇴장 시간은 `out[]` 배열에 저장
- 각각 오름차순 정렬

```java
Arrays.sort(in);
Arrays.sort(out);
```

---

### 🔹 2단계 – 투포인터로 스위핑

- 포인터 `i`는 입장 배열, `j`는 퇴장 배열
- 다음 시각 `t`는 `min(in[i], out[j])`로 결정
- `[TE, TX)` 조건 때문에 같은 시각에 입장/퇴장이 같이 있으면 퇴장을 먼저 처리

```java
// t에서 퇴장 처리
while (j < N && out[j] == t) {
    cur--;
    j++;
}
// t에서 입장 처리
while (i < N && in[i] == t) {
    cur++;
    i++;
}
```

---

### 🔹 3단계 – 구간 [t, nextT)에서의 모기 수 기록

- 이벤트 처리 후 다음 이벤트 시각을 `nextT`로 잡으면 구간 `[t, nextT)` 동안 `cur`명이 유지
- 이 구간을 이용해 최댓값과 구간을 갱신

```java
if (cur > max) {
    max = cur;
    bestL = t;
    bestR = nextT;
} else if (cur == max && max > 0) {
    if (bestR == t) bestR = nextT; // 연속 구간이면 확장
}
```

---

### 💻 핵심 로직

- 이벤트 시각 `t`를 잡고
- `t`에서 퇴장 먼저 처리 → 입장 처리
- 다음 이벤트 시각 `nextT`를 구해 `[t, nextT)`에서의 `cur`을 최대값과 비교

---

### ⏳ 복잡도 분석

- **시간 복잡도:** O(N log N)
    - 입장/퇴장 배열 정렬: O(N log N)
    - 투포인터 스위핑: O(N)

- **공간 복잡도:** O(N)
    - `in[]`, `out[]` 배열 저장
