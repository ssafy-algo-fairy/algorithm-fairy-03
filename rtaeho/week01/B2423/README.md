### 📌 문제 정보

- **번호:** 2423  
- **제목:** 전구를 켜라  
- **난이도:** Gold 1  
- **분류:** 그래프 최단거리, 다익스트라, 구현

---

### 💡 접근 방식

> 타일(`/`, `\`)은 두 꼭짓점을 대각선으로 연결하며, 90도 회전하면 연결되는 대각선이 반대로 바뀝니다.  
> 따라서 **칸**이 아니라 **꼭짓점** 을 기준으로 그래프를 구성하고, 회전 횟수를 비용으로 두어 최단거리를 구했습니다.

---

### 🔹 1단계 – 꼭짓점 그래프 그리기 

- 정점: `(N+1) x (M+1)`개의 꼭짓점 `(r, c)`
- 이동: 꼭짓점에서 대각선 방향으로 인접한 꼭짓점으로 이동
- 한 번의 대각선 이동은 반드시 어떤 타일 1개를 지나가며,
  - 해당 타일이 원하는 대각선(\ 또는 /)으로 연결돼 있으면 비용 0
  - 아니라면 타일을 회전해야 하므로 비용 1
---

### 🔹 2단계 – 다익스트라

- 우선순위 큐 기반 다익스트라로 해결했습니다.
- 시작점: `(0,0)`
- 도착점: `(N,M)`
- `dist[r][c]`에 해당 꼭짓점까지 필요한 최소 회전 수를 저장했습니다.

---

### 💻 핵심 로직

#### 1) 대각선 이동 및 꼭짓점 범위 체크

```java
int nr = r + dr[k];
int nc = c + dc[k];
if (nr < 0 || nr > N || nc < 0 || nc > M) continue;
```

#### 2) 지나가는 타일 좌표 계산 및 비용 결정

```java
int tr = Math.min(r, nr);
int tc = Math.min(c, nc);
if (tr < 0 || tr >= N || tc < 0 || tc >= M) continue;

int add;
if (dr[k] == dc[k]) add = (map[tr][tc] == '\\') ? 0 : 1;
else add = (map[tr][tc] == '/') ? 0 : 1;
```

#### 3) 다익스트라 거리 갱신

```java
int sum = cost + add;
if (sum < dist[nr][nc]) {
    dist[nr][nc] = sum;
    pq.add(new Node(sum, nr, nc));
}
```

---

### ⏳ 복잡도 분석

- 정점 수: `(N+1)(M+1)` ≤ 251,001  
- 간선 수: 정점당 최대 4개 이동  
- **시간 복잡도:** `O(V log V)` (다익스트라)  
- **공간 복잡도:** `O(V)` (dist 배열)

---

### ⚠️ 어려웠던 점

- 처음에는 꼭짓점만으로 맵을 그려서 규칙대로 이동을 구현하면 풀릴 것 같아 단순 구현으로 접근했지만, 단순한 2차원 맵 처리로는 정리가 되지 않았습니다.
- 문제 카테고리를 보고 다익스트라를 찾아 적용했는데, 다익스트라 자체가 오랜만이라 흐름이 익숙하지 않았고 우선순위 큐를 사용하는 방식도 처음엔 어색했습니다.
- 그리고 꼭짓점에서 이동할 때 타일을 계속 같이 생각하는 게 헷갈렸습니다.
- 결국 꼭짓점과 타일을 분리해서 생각하고, 이동 방향에 따라 필요한 대각선(/ 또는 \)을 판단한 뒤 비용을 부여하는 방식으로 정리하면서 해결했습니다.
