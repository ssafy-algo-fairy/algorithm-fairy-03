### 📌 문제 정보

- **번호:** 1256  
- **제목:** 사전  
- **난이도:** Gold 2  
- **분류:** DP, 조합(조합 수), 그리디(문자 선택)

---

### 💡 접근 방식

> `a`가 N개, `z`가 M개 있을 때 만들 수 있는 모든 문자열을 사전순으로 정렬했을 때 K번째 문자열을 구하는 문제입니다.  
> 조합의 개수를 DP로 미리 계산한 뒤, 매 자리에서 `a`로 시작하는 문자열의 개수를 이용해 `a` 또는 `z`를 선택하는 방식으로 해결했습니다.

---

### 🔹 1단계 – DP로 경우의 수 계산

- `dp[i][j]` = `a`가 i개, `z`가 j개 남았을 때 만들 수 있는 문자열 개수
- 점화식:
  - `dp[i][j] = dp[i-1][j] + dp[i][j-1]`
  - `i==0` 또는 `j==0`이면 한 가지뿐이므로 `dp[i][j]=1`

- `K`보다 큰 값은 `K+1`로 처리하여 오버플로우 방지

---

### 🔹 2단계 – K번째 문자열 구성

- 현재 남은 `aCnt`, `zCnt` 상태에서
- 다음 문자를 `a`로 선택했을 때 만들 수 있는 문자열 개수는 `dp[aCnt-1][zCnt]`

1) `dp[aCnt-1][zCnt] >= K`  
   - K번째 문자열은 `a`로 시작하는 구간 안에 존재  
   - `a`를 붙이고 `aCnt--`

2) `dp[aCnt-1][zCnt] < K`  
   - `a`로 시작하는 문자열 `cnt`개를 모두 건너뛰어야 함  
   - `z`를 붙이고 `zCnt--`, `K -= cnt`

---

### 💻 핵심 코드

#### DP 생성

```java
dp[i][j] = dp[i - 1][j] + dp[i][j - 1];
if (dp[i][j] > K) dp[i][j] = K + 1;
```

#### 문자열 구성

```java
int cnt = dp[aCnt - 1][zCnt];
if (cnt >= K) {
    sb.append('a');
    aCnt--;
} else {
    sb.append('z');
    zCnt--;
    K -= cnt;
}
```

---

### ⏳ 복잡도 분석

- **시간 복잡도:** O(NM + (N+M))
    - DP 계산: O(NM)
    - 문자열 구성: O(N+M)

- **공간 복잡도:** O(NM)
    - DP 테이블 저장

---

### ⚠️ 어려웠던 점

- 처음에는 DFS로 직접 문자열을 하나씩 만들어가며 K번째 문자열을 찾으려고 했지만, 가능한 문자열의 개수가 너무 많아 메모리 초과가 발생했습니다.
- 이후 가능한 개수를 DP로 저장해두고 진행했으나, K를 초과하는 경우를 못 찾아서 한참 걸렸습니다.
