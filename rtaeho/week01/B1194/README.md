### 📌 문제 정보

- **번호:** 1194  
- **제목:** 달이 차오른다, 가자.  
- **난이도:** Gold 1  
- **분류:** BFS, 그래프 탐색, 비트마스킹(상태 관리)

---

### 💡 접근 방식

> 열쇠를 획득하면 이동 가능 경로가 달라지기 때문에 단순 2차원 방문 처리로는 해결할 수 없습니다.  
> 따라서 (y, x, 열쇠상태) 를 하나의 상태로 두고 BFS를 수행했습니다.

---

### 🔹 1단계 – 상태 정의 (비트마스킹)

- 열쇠는 `a ~ f` 총 6개 → `2^6 = 64`가지 상태
- `keys`를 비트마스크로 저장:
  - ex) `000001`: a, `000010`: b, ..., `000011`: ab ...
- 방문 배열: `visited[y][x][keys]`

---

### 🔹 2단계 – BFS

- 큐에 `(y, x, keys, dist)` 형태로 저장
- 각 열쇠의 상태가 갱신되므로 dist를 추적

---

### 🔹 3단계 – 이동 처리

- 벽(`#`)이면 이동 불가
- 열쇠(`a~f`)를 만나면 `keys | (1 << (열쇠번호))`로 상태 업데이트
  - or 연산을 하면 열쇠 업데이트 가능
- 문(`A~F`)을 만나면 `(keys & (1 << (문번호)))`로 열쇠 보유 여부 확인 후 이동 가능
  - and 연산을 하면 열쇠 보유 여부 확인 가능

---

### 💻 핵심 코드

#### 열쇠 획득 (비트 추가)

```java
if (cell >= 'a' && cell <= 'f') {
    key = keys | (1 << (cell - 'a'));
}
```

#### 문 통과 (열쇠 있는지 확인)

```java
if (cell >= 'A' && cell <= 'F') {
    int door = 1 << (cell - 'A');
    if ((keys & door) == 0) continue;
}
```

#### 방문 처리 (3차원)

```java
visited[ny][nx][key] = true;
q.offer(new int[]{ny, nx, key, dist + 1});
```

---

### ⏳ 복잡도 분석

- 상태 수: `N * M * 64` (최대 50 * 50 * 64 = 160,000)
- **시간 복잡도:** O(N * M * 64)
- **공간 복잡도:** O(N * M * 64)

---
### ⚠️ 어려웠던 점

- 처음에는 `visited[y][x]`랑 keys[]를 따로 두고 처리하려고 했으나, 열쇠를 얻고 다시 bfs를 진행하면 잘 안됐습니다.
- 그래서 열쇠의 보유 여부와 `visited` 배열을 같이 관리해야한다는 것을 깨달았고, 비트마스킹을 적용했습니다.
- 오랜만의 비트마스킹 연산에서 `|`는 열쇠 추가, `&`는 열쇠 보유 여부 확인도 가물가물했습니다.
