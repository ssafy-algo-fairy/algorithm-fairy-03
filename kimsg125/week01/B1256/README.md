# 🔤 BOJ 1256 - 사전

- 🔗 문제 링크: https://www.acmicpc.net/problem/1256
- 🏷️ 분류: DP(조합), 그리디(사전순 K번째 구성)

---

## 📌 문제 요약
`a`가 `N`개, `z`가 `M`개 있는 문자열들 중에서  
사전순으로 정렬했을 때 **K번째 문자열**을 출력한다.

- 가능한 문자열 개수는 조합 수 `C(N+M, M)` (또는 `C(N+M, N)`)
- K번째가 존재하지 않으면 `-1`

---

## 💡 풀이 아이디어
사전순에서 항상 `'a'`가 `'z'`보다 앞서므로,
매 자리에서 다음을 판단한다.

현재 남은 개수가:
- `a`: N개
- `z`: M개

라고 할 때, 첫 글자가 `'a'`인 문자열의 개수는

- `'a'`를 하나 사용하고 나머지 배치
- `C(N+M-1, N-1)` (또는 `C(N+M-1, M)`)

이다.

따라서:
- `K <= C(N+M-1, N-1)` 이면 현재 자리는 `'a'`
- 아니면 현재 자리는 `'z'`이고,  
  `K -= C(N+M-1, N-1)` 로 “'a'로 시작하는 모든 경우”를 건너뛴다.

이 과정을 `N==0` 또는 `M==0`이 될 때까지 반복하면 K번째 문자열을 만들 수 있다.

---

## ✅ 조합 DP 정의 (오버플로 방지)
조합 수가 매우 커질 수 있으므로, 필요한 것은 “K보다 큰지 작은지” 판정이다.  
그래서 DP를 `MAX = 1_000_000_001`로 캡(cap) 처리한다.

- `dp[n][r] = min(MAX, C(n, r))`

점화식:
- `dp[n][0] = dp[n][n] = 1`
- `dp[n][r] = min(MAX, dp[n-1][r-1] + dp[n-1][r])`

코드에서는 `comb(n, r)`를 재귀 + 메모이제이션으로 구현했다.

---

## 🧱 전체 알고리즘 흐름
1) 조합 DP 테이블을 미리 채운다.
2) 전체 경우의 수 `dp[N+M][M]`이 `K`보다 작으면 `-1`
3) 아니라면 문자열을 한 글자씩 생성:
   - `N==0`이면 남은 건 전부 `'z'`
   - `M==0`이면 남은 건 전부 `'a'`
   - 아니면 `dp[N+M-1][N-1]`을 기준으로 `'a'` 또는 `'z'` 선택

---

## ❗ 시행착오 (DP 미리 채우기 실수)
이 문제는 DP(조합)를 미리 채우는 과정에서  
**필요한 `n, r` 범위를 모두 포함시키지 못해서 몇 번 틀렸다.**

K번째 문자열을 만들 때는 매 단계에서 `C(N+M-1, N-1)` 같은 값이 반복해서 필요하므로,
`n`이 `0..N+M` 범위를 전부 커버하고, 그 안에서 필요한 `r` 값들이 빠짐없이 계산되어 있어야 한다.

➡️ 그래서 최종적으로 `0..N+M` 범위를 기준으로 조합 값이 모두 채워지도록 구성했다.

---

## ⏱️ 복잡도
- 조합 DP 계산: `O((N+M)^2)`
- 문자열 생성: 길이 `N+M` 만큼 반복 → `O(N+M)`

- 시간 복잡도: `O((N+M)^2)`
- 공간 복잡도: `O((N+M)^2)`

---

## 📝 구현 포인트(코드 기준)
- 조합 값은 커질 수 있으므로 `MAX`로 캡 처리(비교만 필요)
- 매 단계에서 `dp[N+M-1][N-1]`로 `'a'` 시작 경우의 수를 계산
- `'z'`를 선택하면 `K`에서 `'a'` 시작 경우의 수를 빼서 범위를 좁힘
- 최종 결과는 `StringBuilder`로 누적 출력
