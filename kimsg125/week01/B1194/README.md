# 🗝️ BOJ 1194 - 달이 차오른다, 가자.

- 🔗 문제 링크: https://www.acmicpc.net/problem/1194
- 🏷️ 분류: BFS, 상태 확장(열쇠), 비트마스크, 3차원 visited

---

## 📌 문제 요약
N×M 미로에서 시작점 `0`에서 출발해 출구 `1`까지의 **최단 거리**를 구한다.

맵 요소:
- `.` : 빈 칸
- `#` : 벽
- `a~f` : 열쇠 (획득 시 보유 상태에 추가)
- `A~F` : 문 (해당 소문자 열쇠가 있어야 통과 가능)
- `0` : 시작
- `1` : 출구

출력:
- 출구까지 도달 가능하면 최단 이동 횟수
- 불가능하면 `-1`

---

## ❗ 시행착오 (메모리 초과)
처음에는 BFS를 하면서도 “열쇠 상태”를 제대로 상태로 관리하지 못해  
경로를 백트래킹처럼 다루거나, 중복 상태가 너무 많이 생겨 **메모리 초과**가 났다.

➡️ 해결:
- 열쇠 보유 상태를 **비트마스크(mask)** 로 표현하고
- `(x, y, mask)`를 하나의 BFS 상태로 두어
- `visited[x][y][mask]`로 중복 탐색을 막는 방식의 BFS로 해결했다.

---

## 💡 풀이 아이디어 (비트마스크 BFS)
열쇠는 최대 6개(`a~f`)이므로 보유 상태는 `2^6 = 64`가지밖에 없다.

따라서 BFS 상태를 다음처럼 정의한다.

- 위치: `(x, y)`
- 열쇠 상태: `mask` (0~63)
- 이동 횟수: `t`

그리고 방문 체크는:
- `visited[x][y][mask] = true`

즉, 같은 칸이라도 **보유한 열쇠가 다르면 다른 상태**로 취급한다.

---

## ✅ 비트마스크 설계
- `a`를 bit 0, `b`를 bit 1, ..., `f`를 bit 5로 사용

열쇠 획득:
- `mask |= 1 << (key - 'a')`

문 통과 조건:
- 문이 `A`라면 필요한 열쇠는 `a`
- `(mask & (1 << (door - 'A'))) != 0` 이어야 통과 가능

---

## 🔁 BFS 진행 로직
1) 시작점 `0`을 찾아 `(x, y)`로 저장 후, 맵에서는 `.`로 바꿔 처리
2) 큐에 초기 상태 `(x, y, mask=0, t=0)` 삽입
3) 큐에서 하나씩 꺼내며:
   - 현재 칸이 `1`이면 `t` 출력하고 종료
   - 4방향 이동 시도
     - 범위 밖 / 벽(#)이면 skip
     - 열쇠면 `nmask`에 반영
     - 문이면 열쇠 없으면 skip
     - `(nx, ny, nmask)`가 방문하지 않은 상태면 큐에 삽입

4) BFS가 끝날 때까지 출구를 못 찾으면 `-1`

---

## 🧠 왜 BFS가 필요한가?
모든 이동의 비용이 1이므로,
BFS는 상태 공간에서 처음 도달하는 `1`이 항상 최단 거리임을 보장한다.

열쇠 상태까지 포함해도 “간선 비용이 동일”하므로 동일하게 BFS가 최단을 보장한다.

---

## ⏱️ 복잡도
상태 수:
- 위치 `N*M`
- 열쇠 상태 `64`

총 상태: `O(N*M*64)`

각 상태에서 4방향 확인:
- 시간 복잡도: `O(N*M*64)`
- 공간 복잡도: `O(N*M*64)` (`visited` 3차원)

(열쇠 종류가 6개로 고정되어 있어 상태 수가 제한적이라 충분히 가능)

---

## 📝 구현 포인트(코드 기준)
- `visited`를 `boolean[N][M][64]`로 선언해 상태 중복을 강하게 차단
- 문 체크는 “현재 mask”로 판단(열쇠는 이동 후 반영)
- 열쇠를 먹으면 같은 칸이라도 새로운 mask로 다시 방문할 수 있도록 설계
